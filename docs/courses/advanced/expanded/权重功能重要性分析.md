## 📊 当前系统如何综合多个分析师报告

### 1. 研究经理（ResearchManagerV2）的综合方式

**当前实现**：
- 输入：`bull_report`（看涨观点）、`bear_report`（看跌观点）
- 方式：通过LLM提示词，让LLM综合两个观点
- 提示词：`"请综合分析看涨和看跌观点做出决策"`
- **没有权重设置**：两个观点在提示词中同等重要

**代码示例**：
```python
prompt = f"""请综合分析 {company_name}（{ticker}）的投资机会：

【看涨观点】
{bull_report}

【看跌观点】
{bear_report}
"""
```

## 💡 权重的重要性分析

### 重要性：⭐⭐⭐⭐（重要）

**理由**：

1. **LLM的局限性**
   - LLM在综合多个报告时，可能无法自动识别哪个报告更重要
   - 如果没有明确指导，LLM可能会平均对待所有报告
   - 这可能导致短线交易时，基本面报告（如果选择了）影响过大

2. **用户意图的明确性**
   - 用户选择不同的分析师组合，本身就体现了不同的权重意图
   - 短线：技术>新闻>社媒
   - 中长线：基本面>板块>风险
   - 但当前系统无法明确传达这个意图给LLM

3. **结果的可控性**
   - 有了权重，用户可以更精确地控制分析结果
   - 可以验证：权重不同，结果是否不同
   - 可以优化：通过调整权重，找到最佳配置

## 🎯 权重如何体现？

### 方案1：在提示词中明确权重（推荐）

**实现方式**：
```python
# 在提示词中明确说明权重
prompt = f"""请综合分析以下报告，做出投资决策：

【重要报告】（权重高）
- 技术分析报告（权重40%）：{market_report}
- 新闻分析报告（权重30%）：{news_report}
- 社媒分析报告（权重30%）：{sentiment_report}

【次要报告】（权重低）
- 基本面分析报告（权重10%，仅供参考）：{fundamentals_report}

请重点关注重要报告，次要报告仅作为补充参考。
"""
```

**优点**：
- ✅ 实现简单，只需修改提示词
- ✅ 不需要改变系统架构
- ✅ 可以立即生效

**缺点**：
- ⚠️ LLM可能不完全遵循权重
- ⚠️ 权重是"软约束"，不是硬约束

---

### 方案2：在报告传递时体现权重

**实现方式**：
- 重要报告：完整传递
- 次要报告：只传递摘要或关键信息

**优点**：
- ✅ 更直接，LLM自然会关注完整报告
- ✅ 减少token消耗（次要报告摘要）

**缺点**：
- ⚠️ 需要实现报告摘要功能
- ⚠️ 可能丢失重要信息

---

### 方案3：多次调用LLM，加权平均

**实现方式**：
- 对每个报告单独调用LLM，得到评分
- 按权重加权平均

**优点**：
- ✅ 权重是硬约束
- ✅ 结果可控

**缺点**：
- ❌ 实现复杂
- ❌ 成本高（多次调用LLM）
- ❌ 可能失去综合判断的优势

---

## 📋 建议的实现方案

### 推荐方案：在提示词中明确权重

**实现步骤**：

1. **在用户选择分析师时，设置默认权重**
   - 短线：技术40%、新闻30%、社媒30%
   - 中长线：基本面50%、板块30%、风险20%

2. **在提示词中明确说明权重**
   ```python
   # 在 ResearchManagerV2、BullResearcherV2、BearResearcherV2 的提示词中
   prompt = f"""
   请综合分析以下报告，做出投资决策：
   
   【报告权重说明】
   - {report1_name}（权重{weight1}%）：{report1_content}
   - {report2_name}（权重{weight2}%）：{report2_content}
   - {report3_name}（权重{weight3}%）：{report3_content}
   
   请根据权重，重点关注高权重报告，低权重报告仅作为补充参考。
   """
   ```

3. **允许用户自定义权重（可选）**
   - 在单股分析页面，增加权重设置
   - 默认使用系统推荐的权重
   - 高级用户可以自定义

## 🎯 是否需要实现？

### 建议：**实现基础权重功能**

**理由**：

1. **权重确实重要**
   - 不同交易风格需要不同的权重
   - 当前系统无法明确传达用户意图

2. **实现成本低**
   - 只需修改提示词
   - 不需要改变系统架构

3. **用户体验好**
   - 用户可以更精确地控制分析结果
   - 可以验证和优化权重配置

4. **可扩展性强**
   - 未来可以增加权重优化功能
   - 可以通过复盘结果自动调整权重
