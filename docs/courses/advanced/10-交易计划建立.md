### 23.1 回顾：什么是交易计划？

#### 23.1.1 交易计划的本质

在开始建立你的交易计划之前，让我们再次回顾什么是交易计划：

```
交易计划 = 一套完整的、可重复执行的投资决策规则

交易计划回答的核心问题：
│
├── 买什么？（选股规则）
├── 什么时候买？（择时规则）
├── 买多少？（仓位规则）
├── 持有期间怎么办？（持仓规则）
├── 什么时候卖？（止盈止损规则）
└── 如何改进？（复盘规则）
```

#### 23.1.2 为什么需要建立自己的交易计划？

```
没有交易计划的后果
│
├── 决策随意
│   ├── 今天凭感觉买
│   ├── 明天听消息卖
│   └── 没有一致性
│
├── 情绪失控
│   ├── 赚钱时贪婪
│   ├── 亏钱时恐惧
│   └── 无法理性决策
│
├── 无法复盘
│   ├── 不知道做对了什么
│   ├── 不知道做错了什么
│   └── 无法改进
│
└── 长期亏损
    ├── "七亏二平一赚"
    └── 成为那70%
```

#### 23.1.3 好的交易计划的特征

| 特征 | 说明 | 反例 |
|------|------|------|
| **明确性** | 规则清晰，没有模糊地带 | "感觉差不多就买" |
| **完整性** | 覆盖交易的各个环节 | 只有选股规则，没有止损规则 |
| **可执行性** | 规则可以具体执行 | "在合适的时候买入" |
| **一致性** | 每次都按同样的规则执行 | 这次止损5%，下次止损10% |
| **可优化性** | 可以根据复盘结果改进 | 规则过于模糊无法评估 |
| **个性化** | 适合自己的性格和情况 | 盲目照搬别人的系统 |

#### 23.2.2 交易计划的六大模块详解

```
交易计划六大模块
│
├── 模块1：选股规则
│   ├── 定义你要买什么样的股票
│   ├── 设置分析配置和权重
│   ├── 明确选股标准和筛选条件
│   └── 系统功能：单股分析、批量分析
│
├── 模块2：择时规则
│   ├── 定义什么条件下可以买入
│   ├── 大盘环境要求
│   ├── 买入信号和确认条件
│   └── 系统功能：大盘分析、行业分析、多空辩论
│
├── 模块3：仓位规则
│   ├── 定义买多少
│   ├── 总仓位控制
│   ├── 单只仓位和分批策略
│   └── 系统功能：模拟交易
│
├── 模块4：持仓规则
│   ├── 定义持有期间如何管理
│   ├── 检视频率
│   ├── 加仓、减仓、换股条件
│   └── 系统功能：持仓分析
│
├── 模块5：止盈止损规则
│   ├── 定义什么时候卖出
│   ├── 止损比例和类型
│   ├── 止盈目标和方式
│   └── 系统功能：风险辩论、止盈止损单
│
└── 模块6：复盘规则
    ├── 定义如何改进
    ├── 复盘频率和内容
    ├── 案例保存
    └── 系统功能：操作分析、案例库
```

```
┌─────────────────────────────────────────────────────────────────────┐
│                      短线交易计划示例                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   【基本信息】                                                       │
│   系统名称：短线突破系统 V1.0                                        │
│   交易风格：短线（1天-1周）                                          │
│   投资理念：顺势而为，快进快出，严格止损                             │
│   风险偏好：稳健偏激进                                               │
│                                                                     │
│   【选股规则】                                                       │
│   ─────────────                                                     │
│   分析配置：市场分析 + 新闻分析 + 社媒分析                           │
│   权重设置：技术面40%、新闻30%、情绪30%                              │
│                                                                     │
│   选股标准：                                                         │
│   • 技术形态：突破形态（箱体突破、平台突破、三角形突破）             │
│   • 量价关系：放量突破，量比>1.5                                     │
│   • 市场热度：当前热点题材相关                                       │
│   • 流动性：日成交额>5000万                                          │
│                                                                     │
│   筛选条件：                                                         │
│   • 系统评分≥75分                                                   │
│   • 多空辩论：看多观点占优                                          │
│   • 风险等级：中等或以下                                            │
│                                                                     │
│   不能买入：                                                         │
│   • ST股票                                                          │
│   • 连续涨停3天以上的股票                                           │
│   • 新股上市30天内                                                  │
│                                                                     │
│   【择时规则】                                                       │
│   ─────────────                                                     │
│   大盘条件：                                                         │
│   • 大盘非明显下跌趋势                                              │
│   • 大盘分析评级不是"强烈看空"                                       │
│                                                                     │
│   买入信号：                                                         │
│   • 信号1：突破前期高点，放量                                       │
│   • 信号2：回调到支撑位，缩量企稳                                   │
│                                                                     │
│   确认条件：                                                         │
│   • 突破当日收盘站稳突破位                                          │
│   • 次日不跌破突破位                                                │
│                                                                     │
│   【仓位规则】                                                       │
│   ─────────────                                                     │
│   总仓位控制：                                                       │
│   • 牛市：80%    • 震荡市：50%    • 熊市：20%                       │
│                                                                     │
│   单只仓位上限：50%                                                  │
│   持股数量：1-3只                                                    │
│   分批策略：分2批买入                                                │
│   • 第1批：60%计划仓位（突破确认时买入）                             │
│   • 第2批：40%计划仓位（回调到支撑位时买入）                         │
│                                                                     │
│   【止盈止损】                                                       │
│   ─────────────                                                     │
│   止损比例：5%（严格执行，不找借口）                                 │
│   止损类型：固定止损 + 移动止损                                      │
│   • 买入后立即设置5%止损单                                          │
│   • 盈利10%后，止损上移到成本价                                     │
│   • 盈利15%后，止损上移到+10%                                       │
│                                                                     │
│   止盈目标：15-20%                                                   │
│   止盈方式：                                                         │
│   • 达到目标价，一次性止盈                                          │
│   • 或跟踪移动止损，让利润奔跑                                      │
│                                                                     │
│   时间止损：                                                         │
│   • 持仓超过5个交易日仍未盈利，考虑卖出                             │
│                                                                     │
│   【持仓规则】                                                       │
│   ─────────────                                                     │
│   检视频率：每日                                                     │
│   使用功能：每日查看持仓分析                                         │
│                                                                     │
│   加仓条件：                                                         │
│   • 突破后回调到支撑位，且仍在上升趋势中                            │
│   • 仓位未达上限                                                    │
│                                                                     │
│   减仓条件：                                                         │
│   • 出现不利消息                                                    │
│   • 技术形态转弱                                                    │
│                                                                     │
│   换股条件：                                                         │
│   • 发现更好的机会，且当前持仓无明显盈利                            │
│                                                                     │
│   【复盘规则】                                                       │
│   ─────────────                                                     │
│   复盘频率：每日收盘后                                               │
│   复盘内容：                                                         │
│   • 当日操作是否符合规则                                            │
│   • 止盈止损是否执行到位                                            │
│   • 有什么需要改进的                                                │
│                                                                     │
│   案例保存：每笔交易完成后立即保存                                   │
│                                                                     │
│   【纪律规则】                                                       │
│   ─────────────                                                     │
│   绝对不做：                                                         │
│   • 不追涨停板                                                      │
│   • 不在熊市重仓                                                    │
│   • 不扛单（触发止损必须执行）                                      │
│   • 不频繁交易（每周不超过5次）                                     │
│                                                                     │
│   必须做到：                                                         │
│   • 买入前必须设置止损单                                            │
│   • 每日复盘                                                        │
│   • 严格按规则执行                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 23.5 如何设计你的交易计划

#### 23.5.1 设计交易计划的步骤

```
设计交易计划的步骤
│
├── 步骤1：确定交易风格
│   ├── 你有多少时间看盘？

### 24.1 为什么交易计划需要持续优化？

#### 24.1.1 交易计划不是一成不变的

```
交易计划需要持续优化的原因
│
├── 原因1：市场在变化
│   ├── 市场风格会轮换
│   ├── 政策环境会变化
│   ├── 新的投资品种出现
│   └── 原有的规则可能失效
│
├── 原因2：你在成长
│   ├── 交易经验在积累
│   ├── 对市场理解在加深
│   ├── 心态在成熟
│   └── 可以处理更复杂的情况
│
├── 原因3：系统需要迭代
│   ├── 初版系统不可能完美
│   ├── 实践中会发现问题
│   ├── 需要不断修正和完善
│   └── 好的系统是"进化"出来的
│
└── 原因4：避免系统僵化
    ├── 市场永远在变
    ├── 一成不变的系统会被淘汰
    └── 保持学习和适应能力
```

#### 24.1.2 优化的误区

```
优化的常见误区
│
├── 误区1：频繁修改
│   ├── 问题：几笔亏损就改系统
│   ├── 后果：系统混乱，无法评估效果
│   └── 正确做法：收集足够样本再决定是否修改
│
├── 误区2：追求完美
│   ├── 问题：总想找到"圣杯"系统
│   ├── 后果：永远在修改，从不执行
│   └── 正确做法：接受系统的不完美
│
├── 误区3：凭感觉修改
│   ├── 问题：没有数据支撑的修改
│   ├── 后果：越改越差
│   └── 正确做法：用数据说话
│
└── 误区4：推倒重来
    ├── 问题：大幅度修改系统
    ├── 后果：失去原有的优势
    └── 正确做法：小步迭代
```

### 24.2 优化的三个来源

#### 24.2.1 来源1：复盘总结

```
从复盘中发现优化点
│
├── 复盘内容
│   ├── 每笔交易的详细分析

│   └── 系统的哪些规则需要改进
│
├── 复盘方法
│   ├── 使用操作分析功能
│   ├── 对照交易计划规则检查
│   └── 记录需要改进的地方
│
├── 优化方向
│   ├── 选股规则是否有效？
│   ├── 择时规则是否准确？
│   ├── 止盈止损是否合理？
│   └── 仓位规则是否得当？
│
└── 案例
    ├── 发现：连续3笔止损都是追高买入
    ├── 分析：买入时机规则不够严格
    └── 改进：增加"不追涨停板"规则
```

#### 24.2.2 来源2：案例库回顾

```
从案例库中发现规律
│
├── 定期回顾
│   ├── 每月回顾本月案例
│   ├── 每季度做综合分析
│   └── 每年做全面总结

├── 分类分析
│   ├── 按盈亏分类：盈利案例 vs 亏损案例
│   ├── 按风格分类：短线案例 vs 中长线案例
│   └── 按市场分类：牛市案例 vs 熊市案例

├── 发现规律
│   ├── 哪类交易成功率高？
│   ├── 哪种情况容易亏损？
│   └── 有什么共同的成功/失败模式？
│
└── 案例
    ├── 发现：在震荡市中，分批建仓的成功率明显高于一次性买入
    ├── 分析：分批建仓能更好地控制成本
    └── 改进：在震荡市中强制分批建仓
```

```
根据市场变化调整系统

│   └── 市场情绪变化

### 24.3 优化的原则和方法

#### 24.3.1 优化的三大原则

```
┌─────────────────────────────────────────────────────────────────────┐
│                         优化的三大原则                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   原则1：小步迭代                                                   │
│   ─────────────                                                     │
│   • 不要大幅度修改系统                                              │
│   • 每次只改一个变量                                                │
│   • 观察效果后再决定是否保留                                        │
│   • 给每次修改足够的验证时间（至少10笔交易）                        │
│                                                                     │
│   正确做法：止损从5%改到6%，观察10笔交易的效果                      │
│   错误做法：同时修改止损、止盈、仓位多个参数                        │
│                                                                     │
│   ─────────────────────────────────────────────────────────────     │
│                                                                     │
│   原则2：数据支撑                                                   │
│   ─────────────                                                     │
│   • 基于案例库的数据分析                                            │
│   • 不要凭感觉修改                                                  │
│   • 用数据说话                                                      │
│   • 记录修改前后的效果对比                                          │
│                                                                     │
│   正确做法：分析20个案例发现止损5%太紧，改为6%                      │
│   错误做法：觉得止损太紧就改，没有数据支撑                          │
│                                                                     │
│   ─────────────────────────────────────────────────────────────     │
│                                                                     │
│   原则3：保持核心                                                   │
│   ─────────────                                                     │
│   • 核心逻辑不轻易改变                                              │
│   • 只调整参数和细节                                                │
│   • 避免频繁推倒重来                                                │
│   • 保持系统的一致性                                                │
│                                                                     │
│   正确做法：保持"趋势跟踪"的核心逻辑，调整均线参数                  │
│   错误做法：今天趋势跟踪，明天价值投资，没有主线                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

    └── 如果新版本不好，可以回退
