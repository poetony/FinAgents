## 📋 扩写总体策略

### 核心理念：可进化的投资系统

**核心价值主张**：
> 通过持续的"计划→选股→择时→执行→持仓管理→风险管理→复盘"循环，帮助用户找出自己的短板，然后改进，然后进行下一轮，持续改进，借助平台打造成为一套可进化的投资系统。

**循环机制说明**（概念性展示，基于现有复盘功能）：

```
┌─────────────────────────────────────────────────────────────┐
│              可进化投资系统的核心循环                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│    ┌──────────┐                                             │
│    │  计划    │ ←─── 基于复盘结果优化交易计划                │
│    └────┬─────┘                                             │
│         │                                                    │
│         ↓                                                    │
│    ┌──────────┐                                             │
│    │  选股    │ ←─── AI多智能体协作分析                      │
│    └────┬─────┘                                             │
│         │                                                    │
│         ↓                                                    │
│    ┌──────────┐                                             │
│    │  择时    │ ←─── AI市场环境判断                          │
│    └────┬─────┘                                             │
│         │                                                    │
│         ↓                                                    │
│    ┌──────────┐                                             │
│    │  执行    │ ←─── AI仓位管理建议                          │
│    └────┬─────┘                                             │
│         │                                                    │
│         ↓                                                    │
│  ┌──────────────┐                                           │
│  │  持仓管理    │ ←─── AI持仓分析决策支持                     │
│  └──────┬───────┘                                           │
│         │                                                    │
│         ↓                                                    │
│  ┌──────────────┐                                           │
│  │  风险管理    │ ←─── AI风险辩论和评估                       │
│  └──────┬───────┘                                           │
│         │                                                    │
│         ↓                                                    │
│    ┌──────────┐                                             │
│    │  复盘    │ ←─── AI复盘分析找出短板                       │
│    │          │     通过评分识别：                           │
│    │          │     • timing_score (时机问题)                │
│    │          │     • position_score (仓位问题)              │
│    │          │     • emotion_score (心态问题)               │
│    │          │     • attribution_score (操作问题)           │
│    │          │     通过报告识别：                           │
│    │          │     • weaknesses (需要改进的点)               │
│    │          │     • suggestions (改进建议)                 │
│    └────┬─────┘                                             │
│         │                                                    │
│         └──────────→ 改进 → 下一轮循环                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**现有复盘功能支持的短板识别**：

| 复盘评分维度 | 对应交易环节 | 如何识别短板 |
|------------|------------|------------|
| `timing_score` (时机评分) | 择时环节 | 评分低 → 时机把握有问题 |
| `position_score` (仓位评分) | 执行/持仓管理环节 | 评分低 → 仓位管理有问题 |
| `emotion_score` (情绪评分) | 风险管理环节 | 评分低 → 心态/纪律有问题 |
| `attribution_score` (归因评分) | 整体操作 | 评分低 → 操作逻辑有问题 |
| `weaknesses` (需要改进的点) | 各环节 | 直接指出具体短板 |
| `suggestions` (改进建议) | 各环节 | 提供针对性改进方案 |

### 核心原则
1. **可进化循环**：强调"计划→执行→复盘→改进→下一轮"的持续循环
2. **短板识别**：通过AI复盘分析找出用户的交易短板
3. **持续改进**：基于复盘结果优化交易计划和系统配置
4. **AI赋能**：每个环节都有AI多智能体协作支持
5. **系统化成长**：从散户到系统交易者的完整路径

### 扩写重点
- ✅ **突出可进化循环机制**：每个环节都要强调如何通过复盘改进
- ✅ **短板识别方法**：详细讲解如何通过AI复盘找出短板
- ✅ **改进策略**：针对不同短板的改进方法
- ✅ **持续优化路径**：如何通过多轮循环持续提升
- ✅ **AI多智能体协作**：每个环节的AI支持
- ✅ **系统配置优化**：如何通过配置优化提升系统表现

  - **通过复盘评分识别短板**（基于现有功能）
    - `timing_score` 低 → 时机把握有问题
      - 买入时机评分低 → 择时环节需要改进
      - 卖出时机评分低 → 持仓管理环节需要改进
    - `position_score` 低 → 仓位管理有问题
      - 仓位不合理 → 执行环节需要改进
      - 分批操作不当 → 持仓管理环节需要改进
    - `emotion_score` 低 → 心态/纪律有问题
      - 情绪控制差 → 风险管理环节需要改进
      - 纪律执行差 → 执行环节需要改进
    - `attribution_score` 低 → 操作逻辑有问题
      - 收益归因不清 → 整体交易计划需要改进
  - **通过复盘报告识别短板**（基于现有功能）
    - `weaknesses` 列表：直接指出需要改进的点
      - 分析weaknesses属于哪个环节
      - 识别重复出现的问题模式
    - `suggestions` 列表：提供改进建议
      - 将建议转化为具体的改进行动
      - 制定改进优先级
  - **改进策略制定**
    - 针对时机短板的改进（timing_score低）
      - 如何改进择时能力
      - 如何利用AI择时分析
    - 针对仓位短板的改进（position_score低）
      - 如何改进仓位管理
      - 如何利用AI仓位建议
    - 针对心态短板的改进（emotion_score低）
      - 如何改进情绪控制
      - 如何严格执行纪律
    - 针对操作短板的改进（attribution_score低）
      - 如何改进操作逻辑
      - 如何优化交易计划
  - **系统配置优化**
    - 如何根据短板调整工作流配置
    - 如何调整AI参数（偏好类型、分析深度）
    - 如何优化分析流程
  - **下一轮循环准备**
    - 改进后的交易计划
    - 改进后的系统配置
    - 改进后的操作流程
  - **实战案例**：展示一个完整的"复盘→通过评分和报告找短板→改进→下一轮"循环
