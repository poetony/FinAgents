# 个人交易计划模块设计概述

> 版本：v2.0  
> 创建日期：2024-12-24  
> 状态：设计中

---

## 1. 模块定位

### 1.1 为什么需要这个模块？

在当前系统中，我们已经具备了完整的交易闭环功能：

```
现有功能覆盖的闭环：
选股 → 择时 → 执行 → 持仓管理 → 风险管理 → 复盘
  │      │      │        │           │         │
  ↓      ↓      ↓        ↓           ↓         ↓
单股分析  大盘分析  模拟交易   持仓分析    风险辩论   操作分析
批量分析  行业分析             止盈止损              案例库
自选股    多空辩论
```

同时，我们的高级课程（`docs/courses/advanced`）教用户如何建立自己的交易计划。

**但存在一个关键缺口：**

> 课程第 23 课让用户写出「个人交易计划」，但系统里没有地方来**承接、存储、应用**这个交易计划。

这导致：
- 用户写出的交易计划只能存在纸上或本地文档
- 系统功能和用户的交易规则是割裂的
- 无法自动对照"用户定义的规则"来检查执行情况

### 1.2 模块核心价值

```
个人交易计划模块 = 把课程产出变成系统内的"活文档"

                    ┌─────────────────────────────────────┐
                    │          个人交易计划模块             │
                    ├─────────────────────────────────────┤
                    │                                     │
                    │   存（承接课程产出）                  │
                    │   ├── 填写/编辑交易计划              │
                    │   ├── 版本化管理                     │
                    │   └── 结构化存储                     │
                    │                                     │
                    │   用（在功能中生效）                  │
                    │   ├── 选股：按系统规则筛选           │
                    │   ├── 下单：带出默认仓位/止损        │
                    │   └── AI分析：注入系统上下文         │
                    │                                     │
                    │   评（复盘时对照）                    │
                    │   ├── 检查执行是否符合系统           │
                    │   ├── 标记违反纪律的操作             │
                    │   └── 支持系统优化迭代               │
                    │                                     │
                    └─────────────────────────────────────┘
```

### 1.3 与现有功能的关系

**当前版本的联动策略：保守渐进**

> 考虑到用户的交易计划在初期可能不够完善，直接用来指导分析和交易决策过于激进。
> 因此当前版本采用**静态保存 + 复盘对照**的方式，不干预用户的实际交易决策。

| 现有功能 | 联动方式 | 说明 |
|---------|---------|------|
| 单股分析/批量分析 | ❌ 暂不联动 | 未来版本考虑 |
| 模拟交易/下单 | ❌ 暂不联动 | 未来版本考虑 |
| 持仓分析 | ❌ 暂不联动 | 未来版本考虑 |
| **操作分析/复盘** | ✅ 联动 | 复盘时对照系统规则，检查执行情况 |

**未来版本可考虑的联动（需用户主动开启）：**
- 分析时显示"系统匹配度"参考
- 下单时提示"系统建议仓位/止损"
- 这些功能应作为"可选参考"，而非强制指导

## 2. 功能范围

### 2.1 核心功能（MVP）

1. **交易计划管理**
   - 创建个人交易计划（向导式填写）
   - 查看/编辑交易计划
   - 版本管理（保存历史版本、切换激活版本）
   - 导出为 Markdown/PDF

2. **系统规则读取**
   - 提供统一接口，供其他模块读取当前激活的交易计划
   - 规则解析与结构化

3. **执行检查**
   - 在复盘时对照系统规则
   - 标记违规操作

### 2.2 扩展功能（后续版本）

- 系统模板库（预设短线/中长线等模板）
- 系统回测（用历史数据验证系统规则）
- AI 辅助优化建议
- 多套系统支持（短线一套、长线一套）

---

## 3. 设计原则

1. **结构化优先**：交易计划存储为结构化数据，便于程序读取和处理
2. **向后兼容**：对现有功能的改动最小化，通过"可选读取"的方式集成
3. **用户友好**：提供向导式填写界面，降低用户理解和使用门槛
4. **版本追溯**：支持版本历史，便于用户对比和回滚
