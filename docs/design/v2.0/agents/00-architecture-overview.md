# 插件化智能体架构设计 v2.0

## 概述

本文档描述 TradingAgentsCN 项目的插件化智能体架构设计，实现 Tools、Agents、Workflows 三层完全解耦，支持灵活的组合和扩展。

## 设计目标

### 核心目标

1. **工具独立化**：每个工具独立定义，可被任意 Agent 引用
2. **Agent 可组合**：Agent 可动态配置使用哪些工具
3. **Workflow 可编排**：Workflow 可动态配置包含哪些 Agent
4. **配置化管理**：所有绑定关系通过配置/数据库管理，无需修改代码
5. **向后兼容**：平滑迁移现有 tradingagents 模块

### 设计原则

- **单一职责**：每个组件只做一件事
- **开闭原则**：对扩展开放，对修改关闭
- **依赖倒置**：高层模块不依赖低层模块，都依赖抽象
- **配置优于代码**：运行时行为由配置决定

## 核心组件

### 1. Tool Layer（工具层）

| 组件 | 职责 |
|------|------|
| `ToolRegistry` | 工具注册中心，管理所有工具的元数据和实现 |
| `BaseTool` | 工具基类，定义工具的标准接口 |
| `ToolFactory` | 工具工厂，根据配置创建工具实例 |
| `@register_tool` | 装饰器，自动注册工具到 Registry |

### 2. Agent Layer（智能体层）

| 组件 | 职责 |
|------|------|
| `AgentRegistry` | Agent 注册中心，管理所有 Agent 的元数据 |
| `BaseAgent` | Agent 基类，定义 Agent 的标准接口 |
| `AgentFactory` | Agent 工厂，根据配置创建 Agent 实例并绑定工具 |
| `@register_agent` | 装饰器，自动注册 Agent 到 Registry |

### 3. Workflow Layer（工作流层）

| 组件 | 职责 |
|------|------|
| `WorkflowRegistry` | Workflow 注册中心，管理所有工作流定义 |
| `WorkflowBuilder` | 工作流构建器，根据配置组装 Agent 为工作流 |
| `WorkflowEngine` | 工作流执行引擎，执行已构建的工作流 |

### 4. Config Layer（配置层）

| 组件 | 职责 |
|------|------|
| `ConfigManager` | 配置管理器，加载和管理所有配置 |
| `ToolAgentBinding` | 工具-Agent 绑定配置 |
| `AgentWorkflowBinding` | Agent-Workflow 绑定配置 |

## 数据流

```
用户请求
    │
    ▼
┌─────────────────┐
│ WorkflowEngine  │ ← 从 ConfigManager 获取 Workflow 配置
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ WorkflowBuilder │ ← 从 AgentRegistry 获取 Agent 列表
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  AgentFactory   │ ← 从 ToolRegistry 获取 Tool 列表
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   Agent 执行    │ ← 调用绑定的 Tools
└────────┬────────┘
         │
         ▼
    分析结果
```

### 5. State Layer（状态层）

| 组件 | 职责 |
|------|------|
| `AgentIODefinition` | Agent 输入输出字段声明 |
| `StateSchemaBuilder` | 根据 Workflow 的 Agent 列表动态生成状态 Schema |
| `StateFieldDefinition` | 状态字段定义（类型、默认值、合并器） |

        ┌───────────────────────────┼───────────────────────────┐

│               │         │               │         │               │
│ 读: input_a   │         │ 读: input_b   │         │ 读: input_c   │
