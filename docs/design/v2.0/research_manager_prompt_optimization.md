# Research Manager v2.0 提示词优化方案

## 📊 问题分析

### 当前问题

通过分析数据库中的提示词模板，发现以下格式冲突：

1. **格式要求重复**：
   - `system_prompt` 包含完整的JSON格式定义（约200字符）
   - `user_prompt` 也包含JSON格式要求（约150字符）
   - `constraints` 也包含JSON格式要求
   - 三个地方都在强调格式，造成冲突和混淆

2. **格式定义不一致**：
   - `output_format` 字段定义的是Markdown格式（研究总结报告格式）
   - 但实际代码期望的是JSON格式输出
   - 格式要求与实际需求不匹配

3. **职责混乱**：
   - `system_prompt` 应该只定义角色和职责，不应该包含格式要求
   - `user_prompt` 应该只提供数据和任务描述，不应该包含格式要求
   - `analysis_requirements` 包含了一些格式相关的关键词（如"买入/持有/卖出"）

### 冲突统计

- `system_prompt`: 包含14个格式关键词
- `user_prompt`: 包含10个格式关键词
- `constraints`: 包含7个格式关键词
- `analysis_requirements`: 包含6个格式关键词
- `output_format`: 包含4个格式关键词（但格式错误）

## 🎯 优化方案

### 原则

1. **单一职责原则**：每个字段只负责一个方面
2. **格式统一**：所有格式要求统一在 `output_format` 字段
3. **清晰分离**：角色定义、任务描述、格式要求、约束条件分离

### 字段职责划分

#### 1. `system_prompt` - 角色定义和职责
**职责**：定义Agent的角色、分析风格、核心职责

**应包含**：
- 角色定位（研究经理）
- 分析风格（激进/中性/保守）
- 核心职责（综合看涨看跌观点、权衡证据、做出决策）
- 决策原则（客观、理性、基于证据）

**不应包含**：
- ❌ JSON格式定义
- ❌ 字段说明
- ❌ 输出格式要求
- ❌ 格式约束

#### 2. `tool_guidance` - 工具使用指导
**职责**：说明如何使用工具或数据

**应包含**：
- 基于提供的分析报告进行综合分析
- 从特定角度（激进/中性/保守）评估信息

**当前状态**：✅ 基本正确，无需修改

#### 3. `analysis_requirements` - 分析内容要求
**职责**：定义分析应该包含哪些内容

**应包含**：
- 多空观点汇总要求
- 综合判断要求
- 投资建议要求
- 目标价格设定原则（逻辑要求，非格式要求）

**不应包含**：
- ❌ JSON格式要求
- ❌ 字段必须性说明
- ❌ 输出格式约束

#### 4. `output_format` - 输出格式要求（核心）
**职责**：统一定义所有格式要求

**应包含**：
- JSON格式定义
- 所有字段说明（action, confidence, target_price, reasoning等）
- 字段类型和约束
- 必需字段和可选字段
- 字段值范围（如confidence: 0-100整数）
- 字段内容要求（如reasoning: 200-500字）

**不应包含**：
- ❌ Markdown格式（当前错误）
- ❌ 分析内容要求（应该在analysis_requirements中）

#### 5. `user_prompt` - 任务描述和数据
**职责**：提供具体的任务描述和输入数据

**应包含**：
- 任务描述（综合分析投资机会）
- 输入数据（股票代码、公司名称、看涨观点、看跌观点等）
- 任务上下文（权重说明、时间上下文等）

**不应包含**：
- ❌ JSON格式要求（应该在output_format中）
- ❌ 字段说明（应该在output_format中）
- ❌ 格式约束（应该在constraints中）

#### 6. `constraints` - 约束条件
**职责**：定义分析立场和基本约束

**应包含**：
- 分析立场约束（激进/中性/保守）
- 语言要求（中文）

**不应包含**：
- ❌ JSON格式要求（应该在output_format中）
- ❌ 字段必须性说明（应该在output_format中）

## 📝 优化后的模板结构

### system_prompt（优化后）

```
你是一位{偏好}的研究经理，需要综合看涨和看跌观点做出决策。

**分析风格**: {分析风格描述}

**核心职责**:
1. 综合分析看涨和看跌观点
2. 权衡双方的理由和证据
3. 做出{偏好}、理性的投资决策
4. 给出明确的投资建议

**决策原则**:
- {根据偏好调整的决策原则}
- 客观、理性、基于证据
- 详细说明决策理由
- 使用中文输出
```

### output_format（优化后）

```
**输出格式要求**：

请严格按照以下JSON格式输出投资计划：

```json
{
    "action": "买入|持有|卖出",
    "confidence": 0-100的整数（必需字段，表示对投资建议的信心度）,
    "target_price": 数字（可选字段，目标价格，必须基于报告中的真实数据）,
    "risk_score": 0-1的数字（可选字段，风险评分）,
    "reasoning": "字符串（必需字段，200-500字，详细的决策理由和分析依据）",
    "summary": "字符串（可选字段，投资计划摘要）",
    "risk_warning": "字符串（可选字段，风险提示）",
    "position_ratio": "字符串（可选字段，建议持仓比例）"
}
```

**字段说明**：
1. **action** (必需): 投资建议，只能是"买入"、"持有"或"卖出"
2. **confidence** (必需): 信心度，必须是0-100之间的整数（如：62、75、80），不是小数
3. **target_price** (可选): 目标价格，必须基于报告中的真实价格数据，严禁编造
4. **reasoning** (必需): 决策理由，200-500字，必须详细说明：
   - 为什么做出这个投资建议
   - 基于哪些分析依据（技术面、基本面、市场环境、看涨观点、看跌观点等）
   - 关键判断因素和逻辑推理过程
5. 其他字段为可选字段，根据需要填写

**重要提示**：
- 必须严格按照JSON格式输出，确保所有必需字段都存在
- confidence必须是整数，不是小数
- target_price必须基于报告中的真实数据，不能随意编造
- reasoning必须详细说明决策理由，不能使用默认值或模板文字
```

### user_prompt（优化后）

```
请综合分析 {{company_name}}（{{ticker}}）的投资机会：

📊 **基本信息**：
- 股票代码：{{ticker}}
- 公司名称：{{company_name}}
- 分析日期：{{analysis_date}}
- 当前价格：¥{{current_price}}

【看涨观点】
{{bull_report}}

【看跌观点】
{{bear_report}}

【辩论总结】
{{debate_summary}}

请基于以上信息，综合分析并给出投资建议。

**权重说明**：
- 看涨观点和看跌观点权重相等（各50%），请同等重视
- 请客观权衡双方观点，基于证据做出理性决策

**⏰ 时间上下文说明**：
- 当前分析日期：{{analysis_date}}
- 如果建议"等待财报"或"等待年报"，请注意：
  * 不要指定具体年份（如"等待2024年年报"）
  * 直接说"等待年报发布"或"等待下一期财报"
  * 或根据当前月份智能判断（1-4月等待上一年度年报，5-12月等待本年度年报）
```

### constraints（优化后）

```
必须从{偏好}角度进行分析，保持一致的分析立场。所有内容使用中文。
```

## ✅ 优化效果

### 优化前
- 格式要求分散在5个字段中
- 存在15个格式冲突点
- 格式定义不一致（Markdown vs JSON）

### 优化后
- 格式要求统一在 `output_format` 字段
- 各字段职责清晰，无冲突
- 格式定义统一为JSON
