## 🎯 背景与需求

### 核心痛点

当前分析报告面向专业用户，包含大量专业术语和技术指标，普通散户难以理解：

```
报告原文（太专业）：
"MACD 死叉，KDJ 进入超卖区域，布林带收窄，
 RSI 回落至 35 附近，成交量持续萎缩..."

用户期望（大白话）：
"简单说，这只股票最近跌得比较多，短期可能还有压力，
 但已经跌到一个相对便宜的位置了，需要观察是否企稳..."
```

### 三类解读需求

| 类型 | 场景 | 示例 |
|------|------|------|
| **单报告通俗解读** | 用户看一份报告，想快速理解核心结论 | "帮我看懂这份贵州茅台的分析报告" |
| **纵向对比** | 同一股票不同时期的报告对比 | "贵州茅台上周 vs 这周有什么变化？" |
| **横向对比** | 同行业不同股票的对比 | "白酒板块里，茅台、五粮液、泸州老窖哪个好？" |

## 🏗️ 系统架构

### 核心服务

```
┌─────────────────────────────────────────────┐
│         ReportInterpretationService          │
├─────────────────────────────────────────────┤
│                                             │
│  模式 1：单报告通俗解读                       │
│    输入：report_id                           │
│    输出：通俗化摘要 + 可追问                   │
│                                             │
│  模式 2：纵向对比                             │
│    输入：stock_code + 时间范围                │
│    输出：变化趋势 + 关键指标对比表             │
│                                             │
│  模式 3：横向对比                             │
│    输入：stock_codes[] (同行业)               │
│    输出：多维度对比表 + 各股优劣势              │
│                                             │
│  模式 4：组合概览                             │
│    输入：user_id (自选股)                     │
│    输出：各股票一句话摘要 + 需关注的变化        │
│                                             │
└─────────────────────────────────────────────┘
```

### 数据流

```
用户操作（前端）
    │
    ▼
API 请求 → app/routers/report_interpretation.py
    │
    ▼
业务逻辑 → app/services/report_interpretation_service.py
    │
    ├→ 从 MongoDB 读取报告内容
    ├→ 提取关键信息（结构化）
    ├→ 构造 Prompt（含合规约束）
    ├→ 调用统一 LLM 客户端
    │
    ▼
返回解读结果 → 前端展示
```

## 📝 功能详细设计

### 功能 1：单报告通俗解读

**前端入口**：报告详情页 → "帮我看懂这份报告" 按钮

**后端流程**：
1. 根据 `report_id` 从 `analysis_reports` 或 `unified_analysis_tasks` 获取报告
2. 提取各子报告的关键结论（市场分析、基本面、技术面等）
3. 构造通俗化 Prompt，将专业术语转为日常语言
4. 调用 LLM 生成通俗解读
5. 返回结构化结果（核心结论、风险提示、关键指标解释）

**Prompt 设计要点**：
```
System Prompt:
- 你是一位面向普通投资者的分析报告解读助手
- 用简单易懂的语言解释专业内容
- ⚠️ 合规约束：不提供买卖建议、目标价、仓位建议
- 对不确定的内容明确标注"报告未涉及"
- 解释专业术语时用类比和日常语言

User Prompt:
- 以下是 {stock_name}({stock_code}) 的分析报告
- 请用通俗语言解读以下内容：
  1. 这只股票目前整体情况如何？（一句话总结）
  2. 主要的利好因素有哪些？（用大白话）
  3. 主要的风险是什么？（用大白话）
  4. 需要重点关注什么？
```

### 功能 2：纵向对比（同股票不同时期）

**前端入口**：
- 报告详情页 → "与历史报告对比"
- 股票详情页 → 选择多份报告 → "对比分析"

**后端流程**：
1. 查询该股票的历史报告（按时间排序）
2. 提取每份报告的关键指标（PE、ROE、技术面评分等）
3. 生成对比 Prompt，要求 LLM 分析变化趋势
4. 输出变化摘要 + 关键指标对比表

**输出格式**：
```
变化趋势摘要：
  "与上周相比，贵州茅台的技术面有所改善：
   - MACD 从死叉转为金叉（利好信号）
   - 成交量放大 20%（资金开始关注）
   - 但基本面评估没有变化（业绩稳定）"

### 功能 3：横向对比（同行业不同股票）

**前端入口**：
- 行业页面 → 选择多只股票 → "横向对比"
- 自选股页面 → 勾选多只 → "对比分析"

**后端流程**：
1. 获取选中股票的最新分析报告
2. 提取各股票的关键维度数据
3. 生成横向对比 Prompt
4. 输出多维度对比表 + 各股优劣势总结

**输出格式**：
```
多维度对比：
| 维度     | 贵州茅台  | 五粮液   | 泸州老窖 |
|---------|----------|---------|---------|
| 基本面   | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐  | ⭐⭐⭐⭐  |
| 技术面   | ⭐⭐⭐    | ⭐⭐⭐⭐  | ⭐⭐⭐   |
| 估值水平 | 偏高      | 合理     | 偏低    |
| 成长性   | 稳定      | 较好     | 较好    |

各股特点：
- 贵州茅台：行业龙头，基本面最强，但估值偏高
- 五粮液：技术面回暖，估值合理，性价比较好
- 泸州老窖：估值最低，成长性不错，但品牌力弱于前两者
```

### 功能 4：组合概览（自选股摘要）

**前端入口**：自选股页面 → "一键解读"

**后端流程**：
1. 获取用户自选股列表
2. 批量获取最新报告摘要
3. 生成组合概览 Prompt
4. 输出各股一句话摘要 + 需关注的变化

## 🔐 合规约束

### 必须遵守的规则

所有解读输出必须遵守以下合规约束（复用现有合规体系）：

| 约束 | 说明 |
|------|------|
| ❌ 不提供买卖建议 | 不说"建议买入/卖出" |
| ❌ 不给目标价 | 不说"目标价 XX 元" |
| ❌ 不给仓位建议 | 不说"可以建仓 30%" |
| ❌ 不做收益承诺 | 不说"预期收益 XX%" |
| ✅ 只做信息解读 | "报告显示...""数据表明..." |
| ✅ 提示投资风险 | 每次解读末尾附风险提示 |

### Prompt 中的合规嵌入

```python
COMPLIANCE_CONSTRAINT = """
⚠️ 合规要求（必须严格遵守）：
1. 你是信息解读助手，不是投资顾问
2. 只解释报告中已有的内容，不做超出报告的推断
3. 不得提供任何买卖建议、目标价、仓位建议
4. 不得做出任何收益承诺或保证
5. 使用"报告显示""数据表明"等客观表述
6. 在解读末尾附上标准风险提示
"""
```

## 🧪 LLM 选型建议

| 解读类型 | 推荐模型 | 理由 |
|---------|---------|------|
| 单报告解读 | GPT-4o / Claude Sonnet / DeepSeek-V3 | 信息提取+改写，文本生成模型足够 |
| 纵向对比 | 同上 | 对比总结，不需要复杂推理 |
| 横向对比 | 同上 | 多维度对比，结构化输出 |
| 追问对话 | 同上 | 上下文理解+回答 |

**结论**：不需要推理 LLM（o1/DeepSeek-R1），文本生成模型即可满足所有解读需求。

## 📊 开发优先级

```
第一阶段（最快见效）：单报告通俗解读
  → 后端 1-2 天 + 前端 1-2 天
  → 用户价值最高，改动最小

第二阶段：纵向对比
  → 后端 1-2 天 + 前端 2-3 天
  → 需要历史报告检索逻辑

第三阶段：横向对比
  → 后端 2-3 天 + 前端 2-3 天
  → 需要多报告聚合和对比展示

第四阶段：组合概览 + 追问对话
  → 后端 2-3 天 + 前端 2-3 天
  → 需要对话状态管理
```
