## 🎯 背景

本文档评估 **Agent Skill（智能体技能/Function Calling）** 技术是否适合引入 TradingAgents-CN Pro 项目的核心分析流程。

### Agent Skill 技术简介

Agent Skill 是指让 LLM 具备自主调用工具的能力，主要包括：

- **Function Calling** (OpenAI): LLM 决定调用哪些函数
- **Tool Use** (Anthropic): LLM 自主选择和使用工具
- **动态技能组合**: 根据任务动态编排能力

```python
# Agent Skill 模式示例
response = llm.chat(
    messages=[{"role": "user", "content": "分析300750的买入时机"}],
    tools=[get_stock_data, get_news, calculate_indicator],
    tool_choice="auto"  # LLM 自主决定调用哪些工具
)
```

## 📊 项目需求特点分析

### 当前业务场景

| 特点 | 描述 |
|------|------|
| **场景固定** | 股票分析、持仓分析、复盘分析 |
| **流程稳定** | 分析师 → 研究员 → 辩论 → 交易员 → 风险评估 |
| **输出结构化** | 固定格式的分析报告，非对话式 |
| **一致性要求高** | 同样输入 → 类似结构的输出 |
| **批量处理** | 支持批量分析、定时分析 |

```
├── 报告聚合器 ✅ - 结构化输出

```

## 🔍 适用性对比

| Agent Skill 适合的场景 | 本项目实际需求 | 匹配度 |
|------------------------|---------------|--------|
| 对话式交互，多轮问答 | 生成报告，单次输出 | ❌ 不匹配 |
| LLM 自主决定调用什么工具 | 工具调用顺序已在工作流中定义 | ❌ 不匹配 |
| 动态、不可预测的流程 | 流程固定、可预测 | ❌ 不匹配 |
| 每次输出形式可能不同 | 要求结构化、一致性强 | ❌ 不匹配 |
| 单用户交互式体验 | 批量处理、定时任务 | ❌ 不匹配 |

## ✅ 评估结论

### 核心结论

**不建议** 在核心分析流程中引入 Agent Skill 技术，原因如下：

1. **需求不匹配**: 项目需要的是结构化报告输出，而非对话式交互
2. **流程已固定**: 分析流程已经成熟稳定，不需要 LLM 自主决策执行顺序
3. **一致性优先**: 结构化报告需要高一致性，Agent Skill 的自主性反而是缺点
4. **增加复杂度**: 引入后增加系统复杂度，但核心收益有限
5. **现有架构已满足**: v2.0 的工作流引擎 + Agent 注册表已经充分满足需求

### 可能的补充应用场景（非核心）

如果未来需要引入，可考虑以下**辅助功能**：

| 场景 | 描述 | 优先级 |
|------|------|--------|
| **报告问答** | 用户看完报告后的深度追问，Agent 自主决定调用数据源 | 低 |
| **智能数据补全** | 数据源失败时，LLM 自动选择备选方案 | 低 |
| **自定义分析助手** | 对话式的轻量级分析，非核心报告生成 | 低 |
