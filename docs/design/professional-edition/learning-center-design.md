## 1. 现状分析

### 1.1 本地学习中心现状

| 维度 | 现状 |
|------|------|
| **内容存储** | `docs/learning/` 目录的 Markdown 文件 |

## 2. 架构设计：本地 + 云端协作

### 2.1 核心原则

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户体验原则                              │
├─────────────────────────────────────────────────────────────────┤
│ 1. 离线优先：本地应用必须离线可用，不依赖网络                      │
│ 2. 云端增强：联网时同步进度、解锁专业内容                          │
│ 3. 单一来源：用户只需在云端网站注册一次                           │
│ 4. 无缝切换：本地/网站学习进度自动同步                            │
```

## 3. 学习进度同步设计

### 3.1 进度数据结构

```python
# 学习进度记录
{
    "user_id": str,           # 用户ID（来自云端）
    "article_id": str,        # 文章/课程ID
    "progress": float,        # 进度百分比 0-100
    "completed": bool,        # 是否完成
    "last_position": str,     # 最后阅读位置（标题ID）
    "read_time": int,         # 累计阅读时长（秒）
    "completed_at": datetime, # 完成时间
    "synced_at": datetime,    # 最后同步时间
    "local_updated": datetime # 本地更新时间
}
```

### 3.2 同步策略

```
本地操作                         云端同步
    │                               │
    ▼                               ▼
用户阅读文章                    每5分钟尝试同步
    │                               │
    ▼                               ▼
更新本地进度                    对比 local_updated
(localStorage)                  vs synced_at
    │                               │
    ▼                               ▼
标记为未同步                    冲突解决：
    │                            取最新时间戳
    ▼                               │
触发同步队列                        ▼
    │                          更新本地synced_at
    ▼
网络可用时批量上传
```

### 3.3 离线支持

| 场景 | 处理方式 |
|------|----------|
| **离线阅读** | 正常使用，进度存localStorage |
| **离线学习** | 已下载的专业内容可继续学习 |
| **恢复联网** | 自动同步本地积累的进度 |
| **冲突处理** | 以最新时间戳为准 |

### 5.3 进度追踪实现

```typescript
// stores/learning.ts
import { defineStore } from 'pinia'

export const useLearningStore = defineStore('learning', {
  state: () => ({
    progress: {} as Record<string, LearningProgress>,
    syncQueue: [] as string[],  // 待同步的文章ID
    lastSyncTime: null as Date | null
  }),

  actions: {
    // 更新阅读进度
    updateProgress(articleId: string, position: string, percentage: number) {
      const now = new Date()
      this.progress[articleId] = {
        ...this.progress[articleId],
        progress: percentage,
        last_position: position,
        local_updated: now,
        completed: percentage >= 95
      }

      // 保存到localStorage
      this.saveToLocal()

      // 加入同步队列
      if (!this.syncQueue.includes(articleId)) {
        this.syncQueue.push(articleId)
      }
    },

    // 同步到云端
    async syncToCloud() {
      if (this.syncQueue.length === 0) return

      try {
        const licenseStore = useLicenseStore()
        if (!licenseStore.isActivated) return

        await api.syncProgress(this.syncQueue.map(id => this.progress[id]))
        this.syncQueue = []
        this.lastSyncTime = new Date()
      } catch (e) {
        console.warn('进度同步失败，稍后重试')
      }
    }
  }
})
```

## 10. 总结

### 关键决策

| 决策点 | 方案 |
|--------|------|
| **用户系统** | 复用云端网站，本地不单独建用户系统 |
| **内容存储** | 免费内容本地、专业内容云端 |
| **进度同步** | 本地为主、云端备份，支持离线 |
| **认证方式** | License Key，与云端网站共享用户体系 |

### 用户体验目标

1. **零门槛开始** - 不登录也能使用基础功能
2. **无缝升级** - 一键跳转注册，Key激活即用
3. **离线友好** - 断网不影响已有功能
4. **多端同步** - 本地/网站学习进度统一
