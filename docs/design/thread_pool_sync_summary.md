# 线程池同步方案设计总结

## 📋 问题概述

当前同步服务存在以下问题：
1. **不是真正的并行**：使用 `asyncio.gather` 协程并发，遇到阻塞会卡住整个任务
2. **进度更新不及时**：进度计算有问题，显示"40/5799"但进度为0%
3. **任务容易卡住**：单个股票处理超时会影响整个批次
4. **无法取消**：任务卡住后无法及时取消

## 🎯 解决方案

### 核心思路
使用 **ThreadPoolExecutor（线程池）** 实现真正的多线程并行处理，每个股票在独立线程中处理，互不干扰。

### 关键特性

1. **真正的并行处理**
   - 10-20个工作线程同时处理不同股票
   - 线程之间完全独立，互不干扰

2. **实时进度更新**
   - 每处理完一个股票立即更新统计
   - 每5秒更新一次进度到数据库
   - 进度计算准确：`(已处理数 / 总数) * 100`

3. **任务取消支持**
   - 通过 `CancellationToken` 实现线程安全的取消
   - 工作线程定期检查取消状态
   - 取消后立即停止处理新任务

4. **超时保护**
   - 每个任务有300秒超时限制
   - 超时任务标记为失败，不影响其他任务

5. **错误隔离**
   - 单个股票处理失败不影响其他股票
   - 错误信息单独记录，便于重试

## 🏗️ 架构设计

```
ThreadPoolSyncService
├── ThreadPoolExecutor (10-20 workers)
│   ├── Worker 1 → 处理股票A
│   ├── Worker 2 → 处理股票B
│   └── Worker N → 处理股票N
├── Task Queue (任务队列)
├── Progress Tracker (进度跟踪器)
└── Cancellation Token (取消令牌)
```

## 📊 数据流

```
主线程 (Async)
  │
  ├─> 创建线程池 (10-20 workers)
  ├─> 填充任务队列 (股票列表)
  │
  ├─> Worker Threads (并行处理)
  │   ├─> Worker 1: 股票A → API调用 → 保存数据 → 更新统计
  │   ├─> Worker 2: 股票B → API调用 → 保存数据 → 更新统计
  │   └─> Worker N: 股票N → API调用 → 保存数据 → 更新统计
  │
  └─> Progress Tracker (定期更新)
      └─> 每5秒更新进度到数据库
```

## 🔧 核心类

### 1. ThreadPoolSyncService
- 管理线程池生命周期
- 分发任务到工作线程
- 收集结果并更新进度

### 2. ProgressTracker
- 线程安全的进度跟踪
- 实时统计成功/失败数量
- 计算进度百分比

### 3. CancellationToken
- 线程安全的取消机制
- 支持任务随时取消

## 💡 使用示例

```python
# 创建线程池服务
service = ThreadPoolSyncService(
    max_workers=10,
    timeout_per_task=300
)

# 执行同步
stats = await service.sync_in_thread_pool(
    items=symbols,  # 股票列表
    process_func=sync_symbol_func,  # 处理函数
    job_id="financial_sync"  # 任务ID
)
```

## 📈 性能提升

| 指标 | 当前实现 | 线程池实现 | 提升 |
|------|---------|-----------|------|
| 并行度 | 协程并发（易阻塞） | 10-20线程（真并行） | ⬆️ 10-20x |
| 进度更新 | 批次完成后更新 | 实时更新 | ⬆️ 及时性 |
| 任务取消 | 不支持 | 支持 | ✅ 新增 |
| 超时保护 | 无 | 300秒超时 | ✅ 新增 |
| 错误隔离 | 批次失败影响整体 | 单股票失败不影响 | ✅ 改进 |

## ⚙️ 配置参数

```python
THREAD_POOL_CONFIG = {
    "max_workers": 10,              # 线程池大小
    "timeout_per_task": 300,        # 单任务超时（秒）
    "progress_update_interval": 5,  # 进度更新间隔（秒）
    "enable_retry": True,           # 启用重试
    "max_retries": 3                # 最大重试次数
}
```

## 📝 注意事项

1. **数据库连接**：每个线程需要独立的数据库连接
2. **异步函数**：在线程中调用异步函数需要特殊处理
3. **资源管理**：确保线程池正确关闭
4. **线程安全**：所有共享数据必须线程安全
