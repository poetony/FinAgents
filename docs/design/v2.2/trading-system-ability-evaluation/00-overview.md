# 个人能力评估与学习引导设计（v2.2 规划）

> 本文档仅作为 **v2.2+ 功能规划留档**，当前版本（v2.0）**不实现**本功能。
> 功能目标是在有足够历史数据时，为用户提供“能力雷达图 + 学习建议”，帮助其发现提升空间，而非给出简单好坏评判。

---

## 1. 背景与目标

- 已有能力：
  - 用户可以在系统中写出并保存自己的 **个人交易计划**（选股 / 择时 / 仓位 / 持仓 / 风险 / 纪律）。
  - 复盘模块可以对单笔交易进行 **合规检查**（是否符合自己系统）。
- 新增设想：
  - 在一定数据积累后，基于用户的历史交易与合规结果，对其在不同模块的 **执行能力 / 效果能力** 做评估，
  - 用于反馈“哪一块最容易偏离自己的系统”“哪一块系统本身可能需要优化”，并 **引导用户回到对应课程学习和改进**。

**定位**：
- 这是一个 **学习反馈系统**，不是“给用户贴标签”的打分系统；
- 强调：评估的是“**是否按自己系统稳定执行**”以及“**在系统内的效果表现**”，而非“天赋好坏”。

---

## 2. 评估范围与原则

### 2.1 评估范围

- 覆盖模块：
  - 选股（Selection）
  - 择时（Timing）
  - 执行（Execution）
  - 持仓（Position Management）
  - 风险（Risk Management）
  - 纪律（Discipline）

- 两个层面：
  1. **执行能力**：是否按自己写下的规则去做（系统一致性 / 纪律性）。
  2. **效果能力**：在“基本按系统执行”的前提下，实际结果质量（胜率 / R 倍数等）。

### 2.2 设计原则

1. **数据门槛**：只有当样本量达到一定阈值（如最近 30 天或最近 30 笔有效交易）时才启用评估；
2. **解释友好**：不直接展示复杂公式，只以等级/提示 + 简洁说明输出；
3. **学习导向**：每一项评估都尽量给出“推荐回看课程章节/练习建议”，而不是只给分数；
4. **渐进启用**：先做“执行能力评估”，在数据更充足后再逐步加入“效果能力评估”。

---

## 3. 评估维度设计（简要）

> 下面为高层方案，具体指标和公式留待 v2.2 细化。

### 3.1 选股能力

- 数据来源：
  - 每笔开仓标的是否符合选股规则（must / exclude）。
  - 系统内选股 vs 系统外选股的后续表现（如 20 根 K 线的 R 倍数）。
- 评估要点：
  - 执行：有多少标的是“违反自己选股规则买入”的？
  - 效果：系统内选股的平均 R 是否明显优于系统外？
- 输出示例：
  - 「选股模块：大部分交易标的符合你的系统规则，系统内选股表现优于系统外，说明你的选股规则有一定有效性。」
  - 「选股模块：系统内外选股表现差异不明显，建议回看《选股模块》课程，重新审视筛选条件。」

### 3.2 择时能力

- 数据来源：
  - 进场点 vs 后续价格路径（最大不利波动 MAE / 最大有利波动 MFE）。
- 评估要点：
  - 进场后是否普遍“先大幅回撤再上涨”（容易追高）。
- 输出示例：
  - 「择时模块：多数交易在进场后出现较大回撤，说明你存在一定追高倾向，建议回看择时章节，优化进场节奏。」

### 3.3 执行能力

- 数据来源：
  - 实际成交价 vs 计划价的偏差；
  - 有计划但临时取消的交易记录。
- 评估要点：
  - 下单是否严格按计划执行；
  - 是否频繁“计划了但临场犹豫放弃”。
- 输出示例：
  - 「执行模块：多数交易能按计划成交，但存在少量明显追价，建议留意情绪化操作。」

### 3.4 持仓管理能力

- 数据来源：
  - 实际加减仓行为 vs 系统设定的分批/持仓规则。
- 评估要点：
  - 是否按计划分批；
  - 盈利单是否普遍“提前止盈”。
- 输出示例：
  - 「持仓模块：超过一半盈利单在未达到系统目标前就被提前卖出，建议回看持仓管理章节，练习让盈利单多走一段。」

### 3.5 风险管理能力

- 数据来源：
  - 实际亏损 vs 预设止损；
  - 超过止损的“大亏单”比例。
- 评估要点：
  - 是否能在预设止损附近果断执行；
  - 是否存在频繁“放大亏损”的情况。
- 输出示例：
  - 「风险模块：最近 30 笔交易中，有多笔亏损明显超过系统设定止损，建议重点练习止损纪律。」

### 3.6 纪律执行能力

- 数据来源：
  - 纪律规则（不追涨停 / 不满仓 / 不扛单等）的违规统计。
- 评估要点：
  - 哪些纪律最容易被打破；
  - 是否有持续改善趋势。
- 输出示例：
  - 「纪律模块：本月出现 3 次‘追涨停’行为，建议在下单前增加一次自检流程，确认是否违反纪律。」

---

## 4. 评分与展示方式（方向性设想）

1. **雷达图/条形图**：
   - 维度：选股 / 择时 / 执行 / 持仓 / 风险 / 纪律；
   - 数值含义：更倾向于“执行度/稳定度”，而非“天赋好坏”。
2. **文字总结 + 学习建议**：
   - 每个维度给一句话总结 + 一条学习建议；
   - 建议可绑定到高级课程对应章节（如“建议回看第 23 课第 X 小节”）。
3. **时间维度对比**：
   - 支持“本月 vs 上月”的能力变化对比，突出“是否在变好”。

---

## 5. 启用条件与版本规划

- **启用条件（用户侧）**：
  - 最近 N 笔（如 20/30 笔）交易中，有足够多记录了完整系统和复盘数据；
  - 达到门槛前，仅提示“完成一定数量复盘后将为你生成能力雷达图和学习建议”。
- **依赖前置（系统侧）**：
  - v2.0：个人交易计划 + 复盘合规检查已上线；
  - v2.2：在此基础上增加聚合统计和可视化展示。

> 本设计为 v2.2 及以后版本的方向性规划，后续在具体排期前可再细化指标与交互方案。
