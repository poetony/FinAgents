# 定时分析功能 - 快速开始

## ✅ 功能状态

**状态**: 已实现并可用
**权限**: 需要 Pro 版授权
**前端页面**: 设置 → 定时分析配置
**路由**: `/settings/scheduled-analysis`
**分析引擎**: v2.0 统一任务引擎（与单股分析一致）

---

## 🚀 快速开始（3 步）

### 第 1 步：创建自选股分组

**前端页面**: 设置 → 自选股分组管理

1. 点击"创建分组"
2. 输入分组名称（如"科技股"、"金融股"）
3. 添加股票代码
4. 保存分组

### 第 2 步：创建定时分析配置

**前端页面**: 设置 → 定时分析配置

1. 点击"创建配置"
2. 填写配置名称和描述
3. 添加时间段：
   - **开盘前** (9:00): `0 9 * * 1-5`
   - **午盘** (11:30): `30 11 * * 1-5`
   - **收盘后** (15:30): `30 15 * * 1-5`
4. 为每个时间段选择分组
5. 设置分析深度（1-5 级）
6. 保存配置

### 第 3 步：测试配置（可选但推荐）

1. 在配置列表中找到刚创建的配置
2. 点击"测试执行"按钮（绿色按钮）
3. 确认执行第一个时间段
4. 等待几分钟后查看执行历史

**提示**: 测试执行可以验证配置是否正确，无需等待 CRON 时间

### 第 4 步：启用配置

1. 在配置列表中找到刚创建的配置
2. 打开"状态"开关
3. 系统自动注册定时任务
4. 等待到达指定时间，自动执行分析

## 📋 CRON 表达式速查

**格式**: `分钟 小时 日 月 星期`

| 时间 | CRON 表达式 | 说明 |
|------|-------------|------|
| 每天 9:00 | `0 9 * * *` | 开盘前分析 |
| 每天 11:30 | `30 11 * * *` | 午盘分析 |
| 每天 15:30 | `30 15 * * *` | 收盘后分析 |
| 工作日 9:00 | `0 9 * * 1-5` | 周一到周五 |
| 每 2 小时 | `0 */2 * * *` | 每 2 小时执行 |
| 每周一 9:00 | `0 9 * * 1` | 每周一执行 |

**提示**: 使用"预览 CRON"功能查看下 5 次执行时间

## 🎯 推荐配置

### 配置 1: 日内三次分析

```
配置名称: 日内三次分析
时间段:
  1. 开盘前 (9:00) - 快速分析 (1级)
     - 分组: 重点关注
     - 模型: qwen-turbo
  
  2. 午盘 (11:30) - 基础分析 (2级)
     - 分组: 科技股
     - 模型: qwen-plus
  
  3. 收盘后 (15:30) - 深度分析 (4级)
     - 分组: 所有分组
     - 模型: qwen-max
```

### 配置 2: 周度深度分析

```
配置名称: 周度深度分析
时间段:
  1. 每周一 9:00 - 全面分析 (5级)
     - 分组: 核心持仓
     - 模型: qwen-max
```

## 📊 查看执行结果

### 方法 1: 查看执行历史

1. 在配置列表中点击"历史"按钮
2. 查看每次执行的详细信息：
   - 执行时间
   - 分析股票数量
   - 成功/失败数量
   - 任务 ID 列表

### 方法 2: 查看分析报告

1. 前往"分析报告"页面
2. 筛选日期和股票代码
3. 查看详细的分析报告

### 方法 3: 查看通知

1. 启用"完成时通知"选项
2. 分析完成后会收到系统通知
3. 点击通知查看详情

## ⚙️ 高级配置

### 参数优先级

```
时间段配置 > 分组配置 > 全局默认配置
```

**示例**:
- 全局默认深度: 3 级
- 分组"科技股"深度: 4 级
- 时间段"收盘后"深度: 5 级
- **实际使用**: 5 级（时间段配置优先）

### 模型选择建议

| 分析深度 | 快速模型 | 深度模型 | 说明 |
|---------|---------|---------|------|
| 1-2 级 | qwen-turbo | qwen-plus | 快速分析 |
| 3 级 | qwen-plus | qwen-plus | 标准分析 |
| 4-5 级 | qwen-plus | qwen-max | 深度分析 |

## 🧪 测试执行功能

### 什么是测试执行？

测试执行允许您立即执行定时分析配置，而不需要等待 CRON 时间。这对于：
- ✅ 验证配置是否正确
- ✅ 调试分析参数
- ✅ 快速查看分析效果

### 如何使用？

1. 在配置列表中点击"测试执行"按钮
2. 系统会执行第一个启用的时间段
3. 等待几分钟后查看执行历史
4. 查看生成的分析报告

**详细文档**: `docs/features/scheduled_analysis_test_feature.md`

## 🔧 故障排查

### 问题 1: 配置启用后没有执行

**检查**:
1. 确认 CRON 表达式正确
2. 确认时间段已启用
3. 确认分组中有股票
4. 查看后端日志
5. **使用"测试执行"功能验证配置**

### 问题 2: 部分股票分析失败

**原因**:
- 股票代码无效
- LLM 服务异常
- 网络问题

**解决**:
- 查看执行历史中的失败原因
- 检查 LLM 配置
- 重新执行失败的任务

### 问题 3: 执行时间不准确

**原因**:
- 服务器时区设置
- CRON 表达式错误

**解决**:
- 使用"预览 CRON"功能验证
- 检查服务器时区设置（应为 Asia/Shanghai）
