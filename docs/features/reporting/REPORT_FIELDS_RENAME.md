# 报告字段重命名和顺序优化

## 📋 修改概述

**修改日期**: 2026-01-10  
**修改目的**: 优化报告展示，让用户清晰理解从初步分析到最终决策的完整推理过程

---

## 🔄 字段重命名

### 1. `investment_plan` → `💡 初步投资建议`

**来源**: 研究经理（Research Manager）  
**阶段**: 第三阶段（研究团队分析）  
**内容**: 综合多头和空头研究员观点后的初步投资建议  
**特点**: 
- 基于技术面、基本面、市场情绪的综合分析
- 尚未经过风险管理团队评估
- 可能包含较为激进的建议

**修改前**: `📋 投资建议`  
**修改后**: `💡 初步投资建议`

---

### 2. `final_trade_decision` → `🎯 最终分析结果`

**来源**: 风险管理经理（Risk Manager）  
**阶段**: 第六阶段（最终决策）  
**内容**: 综合风险评估后的最终分析结果  
**特点**:
- 经过激进、保守、中性分析师的风险评估
- 包含仓位管理、止损止盈等风险控制措施
- 更加稳健和可执行

**修改前**: `🎯 最终分析结果`  
**修改后**: `🎯 最终分析结果`

---

## 📊 展示顺序优化

### 修改前
报告模块按照对象键的顺序展示（不确定顺序）

### 修改后
按照分析流程的顺序展示：

```
第一阶段：宏观分析（2个）
├── 📊 大盘指数分析
└── 🏭 行业板块分析

第二阶段：分析师团队（4个）
├── 📈 市场技术分析
├── 💰 基本面分析
├── 💭 市场情绪分析
└── 📰 新闻事件分析

第三阶段：研究团队（多空辩论 + 初步建议）
├── 🐂 多头研究员 / 看涨研究
├── 🐻 空头研究员 / 看跌研究
├── 🔬 研究经理分析
└── 💡 初步投资建议 ← 研究经理的初步建议

第四阶段：交易团队（1个）
└── 💼 交易员计划

第五阶段：风险管理团队（风险辩论）
├── ⚡ 激进分析师 / 激进风险观点
├── 🛡️ 保守分析师 / 保守风险观点
├── ⚖️ 中性分析师 / 中性风险观点
├── 👔 投资组合经理
└── ⚠️ 风险评估

第六阶段：最终决策（1个）
└── 🎯 最终分析结果 ← 风险管理后的最终决策
```

---

## 💡 用户体验改进

### 改进前的问题
1. ❌ 两个字段名称相似（"投资建议" vs "最终分析结果"），用户困惑
2. ❌ 展示顺序混乱，看不出分析流程
3. ❌ 不清楚两个建议的区别和关系

### 改进后的优势
1. ✅ 名称清晰区分（"初步投资建议" vs "最终分析结果"）
2. ✅ 按照分析流程顺序展示，逻辑清晰
3. ✅ 用户能理解完整的推理过程：
   - 初步建议 → 基于市场分析
   - 风险评估 → 评估初步建议的风险
   - 最终决策 → 综合风险后的决策

## 🔧 技术实现

### 前端修改文件

1. **`frontend/src/views/Reports/ReportDetail.vue`**
   - 添加 `moduleOrder` 数组定义展示顺序
   - 添加 `orderedModuleNames` 计算属性
   - 修改 `getModuleDisplayName` 函数中的字段名称
   - 修改默认激活模块的逻辑

2. **`frontend/src/views/Analysis/SingleAnalysis.vue`**
   - 修改 `reportMappings` 数组的顺序和名称
   - 更新 `iconMap` 和 `descMap` 中的字段名称和描述

### 后端无需修改
- 后端返回的字段名称保持不变（`investment_plan`, `final_trade_decision`）
- 只在前端展示层修改名称
